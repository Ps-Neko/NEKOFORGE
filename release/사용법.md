# NEKOFORGE 사용법

NEKOFORGE는 **AI가 내 프로젝트를 더 잘 이해하고 작업하게 도와주는 로컬 개발 하네스**입니다.

핵심은 검수가 아닙니다. 먼저 기존 소스, 문서, 테스트, 규칙을 읽어 AI에게 줄 작업 맥락을 만들고, 마지막에만 안전장치로 검증을 붙입니다.

```text
내 프로젝트 읽기 -> 작업 맥락 정리 -> AI 작업 패킷 만들기 -> AI 작업 -> 검증 후 적용
```

## 한 줄로 이해하기

AI에게 그냥 "이 기능 만들어줘"라고 말하면, AI는 프로젝트 구조를 놓치거나 엉뚱한 파일을 고칠 수 있습니다.

NEKOFORGE는 먼저 프로젝트를 읽고 다음을 정리합니다.

- 어떤 파일이 중요한지
- 어떤 테스트를 봐야 하는지
- 어떤 규칙을 지켜야 하는지
- AI에게 어떤 작업 단위로 맡기면 좋은지
- 마지막에 적용해도 안전한지

즉, NEKOFORGE는 **AI 작업용 작업지시서와 안전장치를 함께 만드는 도구**입니다.

## 언제 쓰나요?

- AI에게 기존 프로젝트 수정을 맡기고 싶을 때
- AI가 프로젝트 맥락을 자주 놓쳐서 답답할 때
- 수정할 파일과 테스트를 먼저 정리하고 싶을 때
- Claude, Codex, Cursor 같은 도구에 넘길 프롬프트를 만들고 싶을 때
- AI가 만든 변경을 바로 적용하기 전에 한 번 멈추고 확인하고 싶을 때

## 무엇이 만들어지나요?

NEKOFORGE는 `.harness/` 폴더에 작업 자료를 만듭니다.

| 결과물 | 의미 |
|---|---|
| `context.md` | 프로젝트 구조, 테스트, 위험 파일, 관련 파일 후보 |
| `SPEC.md` | 이번 작업이 왜 필요한지 정리한 문서 |
| `PLAN.md`, `TASKS.md` | 작업을 작은 단위로 나눈 계획 |
| `task-packets/TASK-001.md` | AI에게 넘길 작업 패킷 |
| `worker-runs/...prompt.md` | 역할별 AI 프롬프트 |
| `REPORT.md` | 최종 검증 결과 |
| `decision.json` | 적용 가능 여부를 기록한 판정 파일 |

## 3분 체험

처음에는 실제 프로젝트를 건드리지 말고, NEKOFORGE 안에 있는 demo부터 봅니다.

```bash
git clone https://github.com/Ps-Neko/NEKOFORGE.git
cd NEKOFORGE
npm install
npm run build
```

먼저 생산성 흐름을 봅니다.

```bash
node dist/src/cli/index.js demo productivity --clean
```

이 demo는 다음 흐름을 보여줍니다.

```text
소스 읽기 -> context 만들기 -> task packet 만들기 -> worker prompt 만들기
```

그 다음 안전장치를 봅니다.

```bash
node dist/src/cli/index.js demo safety --clean
```

이 demo는 위험한 AI 변경이 마지막에 차단되는 모습을 보여줍니다.

## 내 프로젝트에서 쓰기

NEKOFORGE는 아직 npm 공개 배포 전 알파입니다. 지금은 로컬에서 빌드한 뒤 `npm link`로 연결해 쓰는 방식이 가장 쉽습니다.

```bash
cd NEKOFORGE
npm link
```

이제 실제로 쓰고 싶은 프로젝트 폴더로 이동합니다.

```bash
cd 내-프로젝트
harness doctor
harness init --preset cli-tool
```

`harness doctor`는 현재 폴더가 NEKOFORGE를 쓰기 좋은 상태인지 점검합니다.

## 기본 사용 순서

예를 들어 "로그인 실패 5회 후 잠금 기능"을 AI에게 맡기고 싶다면 이렇게 진행합니다.

```bash
harness ask "로그인 실패 5회 후 5분 잠금 기능 추가"
harness context
harness spec
harness plan
harness workers init --profile standard
harness packet TASK-001 --dispatch
```

여기까지가 핵심입니다.

이 단계가 끝나면 `.harness/task-packets/TASK-001.md`가 생깁니다. 이 파일이 AI에게 줄 작업지시서입니다.

## AI에게 무엇을 넘기나요?

가장 먼저 볼 파일은 이것입니다.

```text
.harness/task-packets/TASK-001.md
```

이 파일 안에는 다음이 들어 있습니다.

- 이번 작업 목표
- 먼저 봐야 할 관련 파일
- 참고할 테스트
- 실행 가능한 명령
- 지켜야 할 제한
- 역할별 프롬프트 위치
- 다음에 실행할 명령

Claude, Codex, Cursor 같은 도구에 이 내용을 넘기면 AI가 프로젝트 맥락을 더 잘 잡고 작업할 수 있습니다.

## AI 작업 후 확인하기

AI가 코드를 수정했다면 다음 순서로 기록하고 확인합니다.

```bash
harness work TASK-001
harness review
harness gate --task TASK-001
```

결과는 `REPORT.md`에서 봅니다.

```text
PASS                  적용 가능
PASS_WITH_WARNINGS    적용 가능하지만 주의 필요
NEEDS_HUMAN_REVIEW    사람이 확인해야 함
BLOCK                 위험해서 적용 금지
INSUFFICIENT_EVIDENCE 증거가 부족해서 적용 금지
```

## 적용하기

최종 적용은 자동으로 되지 않습니다. 사람이 보고 승인해야 합니다.

```bash
harness apply --approved
```

NEKOFORGE의 원칙은 단순합니다.

```text
AI는 빠르게 돕는다.
NEKOFORGE는 맥락을 정리한다.
마지막 적용은 증거를 보고 사람이 결정한다.
```

## 처음 보는 사람에게 설명한다면

이렇게 말하면 됩니다.

```text
NEKOFORGE는 내 프로젝트를 AI가 작업하기 쉬운 형태로 정리해주는 도구입니다.
소스와 테스트를 읽어 작업 패킷을 만들고,
AI가 수정한 뒤에는 검증 결과를 보고 적용할지 결정하게 해줍니다.
```

## 기억할 것

- NEKOFORGE의 중심은 **소스 기반 AI 개발 생산성**입니다.
- 검증은 중심 기능을 보호하는 **마지막 안전장치**입니다.
- 먼저 `harness packet`으로 AI 작업 패킷을 만들고, 나중에 `harness gate`로 확인합니다.
- 실제 적용은 항상 `harness apply --approved`를 사람이 실행합니다.
